AddAssetBundleAssetInjector(RegisterAttributeService)

Assigns any argument with a [AssetLoad] attribute with a prefab from the given Asset Bundles. The type to load is determined from the argument's type. There are 3 ways to add/use asset bundles within the service:
  • For mods that use a singular asset bundle, add it as a Singleton for the injector to pull from.
  • One way for mods with multiple asset bundles, is to add them as KeyedSingletons within the service. From there, The [AssetLoad("AssetName", "BundleKey")] attribute has an optional argument for the bundle key. This key is the same used to register the bundle as a singleton.
  • Another way for mods with multiple asset bundles, is to create an implementation of IRegistryAssetBundleResolver that can be added as a Singleton to the service. Ensure the type IRegistryAssetBundleResolver is used for the singleton and NOT the implementing class, otherwise the injector will fail to use your resolver. If a resolver is added to the service, it will take precedence over any key defined in the [AssetLoad] attribute

AddBasicInjectors(RegisterAttributeService)

Adds the following injectors to the Service:
  • AssetBundle Asset Injector
  • Prefab Info Injector
  • Tech Type Injector
Some injectors require additional configuration within the service to use properly. This only needs to be done if the above injector is used.
